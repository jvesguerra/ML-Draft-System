# Implementation Plan v2.0.0 - Draft System Improvements

This plan addresses the requested improvements in `improvements/v2.0.0.md` for the Mobile Legends Draft System.

## 1. Research & Analysis

### 1.1 Current State
- `App.jsx` uses uncontrolled inputs for enemy picks, leading to the "not cleared" bug.
- Hero names are sliced to 2 characters for display.
- No support for Bans or Role/Lane selection.
- Recommendation engine only considers current picks, not bans.

### 1.2 Required Changes
- **Frontend (`App.jsx`):**
    - Convert hero inputs to controlled components.
    - Add state for `alliedBans` and `enemyBans`.
    - Implement hero name autocomplete using a hero list fetched from the server.
    - Improve hero card UI to show full names.
    - Add Role/Lane selection dropdowns for allied picks.
- **Backend (`engine/`):**
    - Update `suggestCounters` to exclude banned heroes.
    - Add `suggestBans` logic.
- **MCP Server (`mcp-server/`):**
    - Update `get_team_score` to accept specific role/lane assignments.
    - (Optional) Provide an endpoint to fetch the list of all hero names/IDs for autocomplete.

## 2. Strategy

### 2.1 UI/UX Refactor
- Use a single `HeroInput` component for both teams and bans to ensure consistent behavior (clearing, validation, autocomplete).
- Use a 10-slot ban display (5 Allied, 5 Enemy).
- Add a "Role" selector to each Allied hero card.

### 2.2 Recommendation Logic
- **Bans:** Filter out any hero in `alliedPicks`, `enemyPicks`, `alliedBans`, or `enemyBans` from recommendations.
- **Ban Suggestions:** Identify heroes that counter the current allied team or are high-threat counters to the current suggestions.

### 2.3 Composition Scoring
- If a user selects a specific role/lane for a hero, use that value instead of the default array from the `heroStore` when calculating lane/role coverage.
